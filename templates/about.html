Project Overview

This project implements a complete time series analysis and forecasting system for Bitcoin prices using Facebook's Prophet model and a Flask web application. The goal is to predict Bitcoin's future price movements based on historical OHLCV (Open, High, Low, Close, Volume) data.

Project Components

1. Data Loading & Preprocessing

The system loads Bitcoin historical data from a CSV file and performs the following operations:

  • Parse the Date column as datetime objects
  • Sort data chronologically
  • Handle missing values using forward-fill method
  • Reformat data into Prophet format: ds (date) and y (price)

2. Exploratory Data Analysis (EDA)

Comprehensive visualization of the dataset to understand patterns and characteristics:

  • Historical price trends over time
  • Price distribution and statistics
  • Daily returns analysis
  • Trading volume patterns

3. Prophet Model

The Prophet model is designed specifically for time series forecasting and can handle:

  • Trend: Captures long-term upward or downward movements
  • Seasonality: Identifies yearly and weekly patterns
  • Changepoints: Detects sudden shifts in trend behavior
  • Uncertainty Intervals: Provides confidence bounds for predictions

4. Model Evaluation

The model is evaluated on a held-out test set (last 90 days) using multiple metrics:

  • MAE (Mean Absolute Error): Average magnitude of prediction errors
  • RMSE (Root Mean Squared Error): Penalizes larger errors more heavily
  • MAPE (Mean Absolute Percentage Error): Percentage-based error metric
  • Directional Accuracy: Percentage of correct up/down predictions

5. Flask Web Application

A user-friendly web interface that provides:

  • Interactive dashboard with real-time data visualization
  • Historical price charts and trend analysis
  • 30-day, 60-day, and 90-day forecast generation
  • Model performance metrics display
  • Confidence intervals for predictions
  • Dynamic forecast regeneration based on user input

How It Works

  1. When the application starts, it loads the Bitcoin CSV file
  2. Data is cleaned and formatted for Prophet
  3. Historical price statistics are computed and printed
  4. The Prophet model is trained on the entire historical dataset
  5. Model performance is evaluated on a test set (last 90 days)
  6. Multiple plots are generated and saved for visualization
  7. The Flask server starts and serves the web application
  8. Users can interact with the dashboard to view forecasts
  9. Forecast horizon can be changed (7, 30, 60, 90 days)
  10. The model regenerates forecasts dynamically based on user selection

Running the Application

To run this application:

  1. Install required dependencies: pip install -r requirements.txt
  2. Ensure the Bitcoin CSV file is in the project directory or data folder
  3. Run the Flask app: python app.py
  4. Open your browser and navigate to http://localhost:5000
  5. Interact with the dashboard to explore forecasts
